## 修改一：配置文件加载 `-c` 参数 Bug 修复

**日期**：2026-06-04
**文件**：`src/main.c`
**函数**：`param_inject_config`

### 问题描述

测试用例6（INI 配置文件加载）失败。

#### 现象

- 命令行直接传 IP（`./DnsRelay 10.3.9.6 10.3.9.4`）正常
- 使用 `-c` 指定配置文件时，上游服务器配置丢失，服务器启动失败

#### 根因

`main()` 中 `config_load_file()` 先正确读取了 INI 文件，但随后 `param_inject_config()` 将 `-c` 及其参数也当作上游 DNS IP 注入，覆盖了正确配置。原函数只处理了 `-d` / `-dd`，遗漏了 `-c` 标记。

### 修改内容

```diff
 }else if (!strcmp(argv[i],"-dd")) {
     config_set(LOG_SECTION,KEY_LOG_LEVEL,(T)TRACE);
     i++;
+}else if (!strcmp(argv[i],"-c")) {
+    i += 2; // 跳过 -c 及其后面的配置文件路径参数
 } else {
```

**改动量**：3 行。

## 修改二：pack_deserialize 缺少最小长度校验导致 Segfault

**日期**：2026-06-04
**文件**：`src/dns/dns_resolver.c`
**函数**：`pack_deserialize`

### 问题描述

测试用例9（格式错误报文处理）中，向服务器发送 4 字节（<12 字节）的异常 UDP 报文后，服务器发生 segfault 崩溃。

#### 根因

`pack_deserialize()` 仅检查了 `len > 512`，未检查 `len < sizeof(SectionHeader)`（即 < 12 字节）。当 4 字节报文传入时：

1. 第 679 行将 4 字节缓冲区强制转换为 12 字节的 `SectionHeader*`
2. 第 680 行 `memcpy(&pac->header, header_in_buf, sizeof(SectionHeader))` 从 4 字节缓冲区读取 12 字节 → **越界读**
3. 越界读到的垃圾数据被当作 `qcount` 字段，成为一个巨大的数值
4. `questions_deserialize()` 按此数值尝试解析大量 Question 记录，读入大量非法内存
5. 最终命中未映射内存页 → **SIGSEGV**

### 修改内容

在 `len > 512` 检查之前，增加最小长度校验：

```diff
 int pack_deserialize(const char *raw_pack, int len, DnsPacket **packet) {
+    if (len < (int)sizeof(SectionHeader)) {
+        ex_throw("pack_deseri: size too small");
+        return -1;
+    }
     if (len > 512) {
         ex_throw("pack_deseri: size exceeded");
         return -1;
     }
```

**改动量**：4 行。与已有 `>512` 检查共同形成合法报文区间 `[12, 512]` 字节的守护。

#### 处理流程

`pack_deserialize` 返回 -1 → `pack_recv` 返回 -1 → `server_loop` 输出 `[ERROR] server loop err: pack_deseri: size too small` → 跳出内层收包循环 → 服务器回到 `select` 继续等待下一个请求。服务器不会停止。
